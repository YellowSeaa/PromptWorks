LLM 管理 / 模型成本配置预览

先讲清楚“怎么计费”，再去填数字

这个预览把原本平铺的成本字段拆成了更顺手的顺序：默认人民币，先填单价，只有需要外币时才展开高级设置。 右侧会实时告诉你，这些值最终会怎么算成一次调用的成本。

默认计价单位1,000,000 tokens
默认展示币种CNY
主流程基础信息 → 计费规则 → 高级设置

1. 基础信息

默认按人民币展示，不需要在主流程里想币种和汇率。

最先填写 默认 CNY
主流程默认值 人民币 / 1
说明： 平时只需要填计费单位和单价。只有当模型价格不是人民币时，再去高级设置里切换币种并填写汇率。

2. 计费规则

主表单只保留最常改的三个值：计价单位、输入单价、输出单价。

核心配置 实时预览
表示“每多少 token 计一次价”，常见是 1,000 或 1,000,000。
按计价币种计价，代表输入 token 的价格。
按计价币种计价，代表输出 token 的价格。
换算逻辑单价 × 实际 token 数 ÷ 计价单位
默认币种人民币 CNY，汇率固定为 1
适用场景没有阶梯计费的普通模型
3. 高级设置 只有当模型价格按上下文窗口分档，或需要单独折算时才展开。
可折叠
币种设置 默认关闭
只要这里保持人民币，汇率就固定为 1；切换到其他币种后，才需要维护汇率。
阶梯价格 按需开启
按上下文上限从小到大匹配，命中第一档即生效。
上下文上限 tokens 输入单价 输出单价 操作
保存后会影响模型列表、用量监控和成本统计。
如果只是普通模型，通常只填基础信息和计费规则就够了。

实时预览

这块是最重要的：用户可以在填完后马上知道“一个请求大概多少钱”。默认按人民币展示。

1M 输入成本 CNY 0.40
1M 输出成本 CNY 1.20
示例请求成本 CNY 0.01
当前币种 CNY / 1
示例：12,000 输入 tokens + 4,000 输出 tokens
输入成本 = 12,000 ÷ 1,000,000 × 0.40
输出成本 = 4,000 ÷ 1,000,000 × 1.20
合计 = 0.01 CNY
主流程先看预览，再保存
默认策略普通模型只填基础规则
复杂场景再展开阶梯价格

这版的使用顺序

1
先确认默认人民币 普通场景不用管币种和汇率，系统默认按 CNY 统计成本。
2
再填单价 如果模型只需要固定输入/输出单价，就在这里填完，其他部分可以不看。
3
有特殊规则再展开高级设置 币种、汇率、阶梯价格都放在折叠区，减少第一次使用的压力。
