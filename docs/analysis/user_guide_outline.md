# 分析报告模块用户指南（草案）

> 本文档面向平台使用者，概述分析模块在前端的使用方式。随着功能迭代会持续补充细节。

## 1. 选择分析模块
- 在测试任务创建页面，通过“分析模块”多选框勾选需要在任务结束后自动执行的分析。
- 任务完成后，可在结果页右上角的模块选择器中动态勾选需要展示的分析卡片。
- 支持随时增删模块，未勾选的模块不会占用计算资源。

## 2. 配置模块参数
- 每个模块会显示所需参数的中文名称、填写说明及示例。
- 若参数类型为下拉框或正则表达式，界面会提供可选项或模板，避免输入错误。

## 3. 查看分析结果
- 分析在卡片中展示运行状态、错误提示与最终结果：
  - 表格：展示分析结果的原始数据，字段来源于模块定义。
  - 图表：若模块声明支持 `bar`、`line`、`pie` 等图表，可在卡片内切换维度和图表类型。
  - 洞察：模块可返回文字化总结或建议，帮助快速理解结果。
- 卡片右上角提供“执行分析 / 重新执行”按钮，可在调整参数后重新计算。
- 默认提供的“耗时与 Tokens 概览”模块，可快速了解性能与资源消耗情况。

## 4. 调用 LLM 的模块
- 支持 LLM 的模块会在卡片上展示调用次数与费用估算。
- 平台会自动控制调用限额并提供失败重试机制，用户无需额外操作。

## 5. 自定义模块（预告）
- 后续版本将支持上传自定义模块并在界面中启用。
- 届时会提供模块模板、参数说明与最佳实践指南，帮助用户快速构建扩展分析。
