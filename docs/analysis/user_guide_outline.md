# 分析报告模块用户指南（草案）

> 本文档面向平台使用者，概述分析模块在前端的使用方式。随着功能迭代会持续补充细节。

## 1. 选择分析模块
- 在测试任务创建页面，通过“分析模块”多选框提前勾选需要的分析。
- 若任务已完成，可在结果页点击“开启分析”按钮，按需选择模块并填写参数。

## 2. 配置模块参数
- 每个模块会显示所需参数的中文名称、填写说明及示例。
- 若参数类型为下拉框或正则表达式，界面会提供可选项或模板，避免输入错误。

## 3. 查看分析结果
- 分析完成后会以卡片形式展示：
  - 表格：展示分析结果的原始数据。
  - 图表：若模块声明支持图表，系统会自动渲染对应的可视化。
  - 洞察：模块可能给出文字化结论或注意事项。
- 支持刷新或重新运行模块，以便在手动调整参数后对比结果。
- 默认提供的“耗时与 Tokens 概览”模块可以帮助你快速了解任务的性能与资源消耗情况。

## 4. 调用 LLM 的模块
- 支持 LLM 的模块会在卡片上展示调用次数与费用估算。
- 平台会自动控制调用限额并提供失败重试机制，用户无需额外操作。

## 5. 自定义模块（预告）
- 后续版本将支持上传自定义模块并在界面中启用。
- 届时会提供模块模板、参数说明与最佳实践指南，帮助用户快速构建扩展分析。
