# Changelog

## [0.7.1] - 2026-06-09

### Added
- 测试任务结果页合并 AI 评分与优化建议入口，保留评分/优化两个核心操作
- 新增 AI 优化工作台页面，支持基于评分摘要查看问题归因、生成 Prompt 改写、人工编辑并创建新版本
- Prompt 改写结果支持一键新增为 Prompt 管理中的新版本，创建后自动激活并跳转 Prompt 详情

### Changed
- Prompt 测试任务只归属于一个 Prompt，后端会拒绝跨 Prompt 版本混入同一测试任务或追加单元
- 顶部导航版本号更新为 v0.7.1
- README 与英文版 README_en 补充 AI 评分与优化工作台作为核心能力亮点

### Fixed
- 修复前端 Docker 多架构构建在 arm64 QEMU 环境中执行 Vite 构建导致 Illegal instruction 的问题

## [0.7.0] - 2026-06-09

### Added
- Prompt 测试任务新增 AI 评分能力，可在创建任务时开启，也可在结果页对既有任务补跑或重跑评分
- 测试输出产出后会立即触发评分，评分状态独立于测试任务状态展示，单条评分失败不会影响测试任务完成
- 测试结果页的最小测试单元卡片新增平均分展示，单元详情页新增每条测试结果的 AI 评分展示
- 顶部导航新增当前版本号展示
- 结果页新增评分徽标、悬浮评分明细、失败重试、评分进度、单元统计和手动优化推荐入口
- AI 优化推荐会基于评分明细、单元统计、Prompt、模型与参数生成参数建议、模型建议和 Prompt 迭代草稿
- 重新评分会持续写入可见进度，刷新页面后仍显示评分中并禁用重复触发按钮
- 运行超时设置新增 AI 优化建议超时时间

### Changed
- 测试结果页优化 AI 评分卡片布局，仅保留一个状态化操作按钮
- AI 评分提示词统一要求 0-100 百分制，并在入库前校验疑似 5 分制或 10 分制响应
- AI 评分与测试执行按模型并发上限并行调度
- 前端构建新增依赖分包策略，拆分 Vue、Element Plus、ECharts 等公共依赖

### Fixed
- 修复异步 AI 评分未预创建待评分记录导致结果页评分数量少于测试输出、进度误显示为已完成的问题
- 修复测试任务完成但 AI 评分仍有待处理记录时，结果页按钮误显示"开始评分"的问题
- 修复单元详情页变量筛选下拉框宽度过窄的问题
- 修复 AI 优化推荐在模型读超时时直接展示底层英文错误的问题
- 补充 LLM 上下文截断与分析模块注册表单元测试
